# The setup wizard creates default environments but no API keys for them

`generateApiKey()` returns the plaintext `fullKey` alongside the values actually persisted (`apiKeyId`, `apiKeyHash` — see docs/adr/0008-api-key-prefix-is-cosmetic-only.md); the plaintext only ever exists for that one request. The setup wizard (`apps/dashboard/src/app/(public)/setup/actions.ts`) used to generate a "Default" key for each of the two environments it creates, but discarded `fullKey` and redirected straight to `/dashboard` — so those keys were hashed into the database and immediately unrecoverable, with no way for the owner to ever copy them. We removed that key creation entirely rather than surface the plaintext post-setup: `apps/api/src/routes/apiKeys.ts`'s create-key route already returns `fullKey` once, and the dashboard's `ApiKeyReveal` component already exists to show it — reusing that normal flow is simpler than adding a one-time reveal step to setup. If a future change adds default keys back, it must show the plaintext to the user in that same request; storing only the hash is useless.
